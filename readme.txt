一、用户登录
    后端实现：
        整体思路：
            Controller层：
                1、通过控制器方法形参接收客户端传递过来的参数（账户名、账户密码）
                2、通过调用业务逻辑层的登录方法，得到登录结果
                3、响应数据给客户端

            Service层：
                1、判断控制层传递过来的账号与密码是否为空，如果参数为空则抛出异常（异常被控制层捕获并处理）
                2、将密码进行加密
                3、调用数据库访问层通过账号与密码查询用户记录返回用户对象
                4、判断用户对象是否为空，如果用户对象为空则抛出异常（异常被控制层捕获并处理）
                6、判断账号是否有效（1为有效，0为无效）
                7、判断控制层传递过来的账号与数据库中查询出的账号是否相同，如果不相同则抛出异常（异常被控制层捕获并处理）

            Dao层：
                通过业务逻辑层传递过来的账号与密码查询用户记录，返回用户对象

二、修改密码：
    后端实现：
        整体思路：
            Controller层：
                1、通过控制器方法形参接收客户端传递过来的参数（旧密码、新密码、确认密码）
                2、通过session获取id
                2、通过调用业务逻辑层的修改密码方法，得到处理结果
                3、响应数据给客户端

            Service层：
                1、判断控制层传递过来的所有是否为空，如果为空则抛出异常（异常被控制层捕获并处理）
                2、判断新密码与确认密码是否相同，如果不相同则抛出异常（异常被控制层捕获并处理）
                3、将用户id进行解密
                4、调用数据库访问层通过用户id查询用户记录返回用户对象
                5、判断用户对象是否为空
                6、将旧密码进行加密后与数据库中查询出的密码进行比较，如果不相同则抛出异常（异常被控制层捕获并处理）
                7、将新密码进行加密后与数据库中查询出的密码进行比较，如果相同则抛出异常（异常被控制层捕获并处理）
                8、将新密码进行加密
                9、调用数据库访问层进行修改密码并返回结果

            Dao层：
                1、通过业务逻辑层传递过来的用户id和新密码进行密码修改操作，返回结果。
